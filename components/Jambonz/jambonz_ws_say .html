# Jambonz Say https://docs.jambonz.org/verbs/verbs/say This component adds a Jambonz `say` verb to the websocket flow using the `tag`-based `flowPath` pattern for sequential verb chaining. ## Inputs / Outputs - **Input** — Receives filtered messages from `jambonz_ws_router` FLOW output - **Output** — Pass-through for messages where this node has already been visited (connect to next flow node) - **debug** — Formatted debug/error messages (when "Send to Debug output" options are enabled) Ack responses are sent directly to the Jambonz websocket client via `$.refs.client` (attached by the router). ## flowPath Pattern Each node self-selects whether to handle or pass through based on the `flowPath` array in the incoming `tag` data: 1. **No flowPath / empty** (first call, `session:new`): Responds with `answer` + `say` verbs wrapped in an ack, including a `tag` verb with `flowPath: [nodeId]` and `step_num: 0` 2. **flowPath exists, node NOT in it** (subsequent `verb:hook`): Appends its nodeId to `flowPath`, increments `step_num`, responds with `say` verb in ack 3. **flowPath exists, node IS in it** (already visited): Passes message through to Output unchanged ## Configuration - **Prompt** (required): Text to speak (supports static text and dynamic templates) - **Vendor** (optional): TTS vendor to use - **Language** (optional): Language code for TTS - **Voice** (optional): Specific voice to use - **Loop**: Number of times to repeat (default: 1) ## Dynamic Value Templates The Prompt field supports template syntax to read from input data and perform math operations. ### Template Syntax: - **Static text**: `Welcome to our service` - **Dynamic path**: `Hello {{ $data.userName }}` - **Math**: `Balance: {{ $data.balance - $data.amount }}` ## Debug Configuration Enable **Enable component debug** to activate diagnostic logging. Options: - **Send Name and ID** — Include component name and unique instance ID in log lines - **Timestamp (ISO 8601)** — Prepend timestamp in the configured timezone - **Timezone** — UTC (default), GMT offsets, or IANA identifiers (with DST) - **JSON output** — Structured JSON format instead of plain text - **Send debug to Debug output** — Route debug messages to the Debug output instead of console - **Send errors to Debug output** — Route error messages to the Debug output instead of console ## Wiring Example ``` Router [FLOW] --> say1 [Output] --> say2 [Output] --> hangup ``` Each node sends its ack directly to the websocket client — no wiring back to the router's Send input needed.
