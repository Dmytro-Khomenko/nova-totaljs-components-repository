# Jambonz Hangup https://docs.jambonz.org/verbs/verbs/hangup This component adds a Jambonz `hangup` verb to the flow data. ## Configuration - **Custom SIP Headers** (optional): Key-value pairs for custom SIP headers to include in the hangup message ## Dynamic Value Templates Both header keys and values support template syntax to read from input data, perform math operations, and concatenate text. ### Example Input Data: ```json { "sessionId": "abc123", "duration": 120, "reason": "completed", "account": { "id": "12345" } } ``` ### Template Syntax: - **Static value**: `X-Custom-Header` → outputs `"X-Custom-Header"` - **Dynamic path**: `{{ $data.sessionId }}` → reads from `$.data.sessionId` → outputs `"abc123"` - **Nested path**: `{{ $data.account.id }}` → reads from `$.data.account.id` → outputs `"12345"` ### Text Concatenation: - `X-Session-{{ $data.sessionId }}` → `"X-Session-abc123"` - `{{ $data.reason }}-call` → `"completed-call"` - `session_{{ $data.sessionId }}_{{ $data.account.id }}` → `"session_abc123_12345"` ### Math Operations: - `{{ $data.duration + 10 }}` → `130` - `{{ $data.duration - 20 }}` → `100` - `{{ $data.duration * 2 }}` → `240` - `{{ $data.duration / 2 }}` → `60` ### Error Handling: Unsupported expressions return `""`: - `{{ $data.nonexistent }}` → `""` (path doesn't exist) - `{{ $data.sessionId + 5 }}` → `""` (cannot add number to string) - `{{ $data.account.invalid }}` → `""` (invalid nested path) ## Behavior - If `jambonz_response` array doesn't exist in input data, creates a new array with the hangup verb - If `jambonz_response` array exists, appends the hangup verb to the existing array - Headers are only added if both key and value are provided - Template expressions are resolved for both header keys and values before building the hangup verb - Invalid template expressions return `""`
