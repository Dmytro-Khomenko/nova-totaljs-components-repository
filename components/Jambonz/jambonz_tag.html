# Jambonz Tag https://docs.jambonz.org/verbs/verbs/tag This component adds a Jambonz `tag` verb to the flow for adding custom metadata/tags to calls. ## Configuration Each tag contains: - **Key**: The key name for the tag data - **Value**: The value for the tag data (supports static values and dynamic templates) - **Type**: Type conversion to apply (string, number, boolean) ## Dynamic Value Templates Values support template syntax to read from input data, perform math operations, and concatenate text. ### Example Input Data: ```json { "key1": "value1", "key2": { "key21": "value21" }, "key3": [ {"key31": "value31"}, {"key32": "value32"} ], "key4": 10 } ``` ### Template Syntax: - **Static value**: `SomeText` → outputs `"SomeText"` - **Dynamic path**: `{{ $data.key1 }}` → reads from `$.data.key1` → outputs `"value1"` - **Nested path**: `{{ $data.key2.key21 }}` → reads from `$.data.key2.key21` → outputs `"value21"` - **Array path**: `{{ $data.key3[0].key31 }}` → reads from `$.data.key3[0].key31` → outputs `"value31"` ### Math Operations: - `{{ $data.key4 + 5 }}` → `15` - `{{ $data.key4 - 3 }}` → `7` - `{{ $data.key4 * 2 }}` → `20` - `{{ $data.key4 / 2 }}` → `5` ### Text Concatenation: - `{{ $data.key1 }} test1` → `"value1 test1"` - `{{ $data.key1 }}_test2` → `"value1_test2"` - `{{ $data.key1 }}-test3` → `"value1-test3"` - `{{ $data.key1 }}2345` → `"value12345"` - `prefix_{{ $data.key1 }}_suffix` → `"prefix_value1_suffix"` ### Error Handling: Unsupported expressions return `""`: - `{{ $data.nonexistent }}` → `""` (path doesn't exist) - `{{ $data.key1 + 5 }}` → `""` (cannot add number to string) - `{{ $data.key2.invalid }}` → `""` (invalid nested path) ## Usage 1. Click "Add tag" to create a new tag entry 2. Fill in Key (key name) and Value (static or template) 3. Select the Type for value conversion (defaults to "string") 4. Add more tags as needed ## Type Conversions - **string**: Converts value to string (default) - **number**: Converts value to number (returns 0 if conversion fails) - **boolean**: Converts value to boolean - accepts: true/false, 1/0, yes/no - returns `false` if conversion fails ## Behavior - If `jambonz_response` array doesn't exist in input data, creates a new array with the tag verb - If `jambonz_response` array exists, appends the tag verb to the existing array - Multiple tags are merged into a single data object within the tag verb - If Key is empty in a tag, that tag is skipped - If duplicate keys exist, the last one wins - Type conversion is applied after template resolution and expression evaluation - Supports multiple templates in a single value (concatenation) - Math operations support: `+`, `-`, `*`, `/` (division by zero returns 0) - **Error handling**: All unsupported expressions return `""`: - Non-existent data paths (e.g., `{{ $data.nonexistent }}`) - Invalid math operations (e.g., `{{ $data.key1 + 5 }}` where key1 is a string) - Any expression that cannot be properly evaluated
