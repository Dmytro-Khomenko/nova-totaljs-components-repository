### Jambonz Gather https://docs.jambonz.org/verbs/verbs/gather This component adds a Jambonz `gather` verb to the websocket flow using the `tag`-based `flowPath` pattern for sequential verb chaining. ## Inputs / Outputs - **Input** — Receives filtered messages from `jambonz_ws_router` FLOW output - **Output** — Pass-through for messages where this node has already been visited (connect to next flow node) - **debug** — Formatted debug/error messages (when "Send to Debug output" options are enabled) Ack responses are sent directly to the Jambonz websocket client via `$.refs.client` (attached by the router). ## flowPath Pattern Each node self-selects whether to handle or pass through based on the `flowPath` array in the incoming `tag` data: 1. **No flowPath / empty** (first call, `session:new`): Responds with `answer` + `gather` verbs wrapped in an ack, including a `tag` verb with `flowPath: [nodeId]` and `step_num: 0` 2. **flowPath exists, node NOT in it** (subsequent event): Appends its nodeId to `flowPath`, increments `step_num`, responds with `gather` verb in ack 3. **flowPath exists, node IS in it** (already visited): Passes message through to Output unchanged Since `gather` has its own `actionHook`, no trailing gather verb is appended (unlike the say component). If this node is the last in the chain (nothing connected to Output), a `hangup` verb is appended after the gather verb. ## Configuration ### Input Settings - **Speech / DTMF** — Input modes with bargein options - **Finish on key** — DTMF terminator character - **No input timeout** — Seconds to wait for input ### Recognizer Settings - **Vendor / Language** — Speech recognition provider and language - **VAD** — Voice Activity Detection with sensitivity and min speech duration - **Custom settings** — Additional recognizer JSON (hints, boosts, etc.) ### Audio Settings - **Audio Type** — Select audio mode: None (default), Play (audio file URL), or Say (text-to-speech) ### Hook Settings - **Action Hook** — URL to send gather results (default: `/transcriber`) - **Hook waiting** — Retry and timeout settings for the action hook response ## Debug Configuration Same debug options as the WS router and say components.
