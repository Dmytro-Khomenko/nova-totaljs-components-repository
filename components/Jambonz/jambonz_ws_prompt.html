# Jambonz Prompt https://docs.jambonz.org/verbs/verbs/say https://docs.jambonz.org/verbs/verbs/play This component adds a Jambonz `say` or `play` verb to the websocket flow using the `tag`-based `flowPath` pattern for sequential verb chaining. ## Inputs / Outputs - **Input** — Receives filtered messages from `jambonz_ws_router` FLOW output - **Output** — Pass-through for messages where this node has already been visited (connect to next flow node) - **debug** — Formatted debug/error messages (when "Send to Debug output" options are enabled) Ack responses are sent directly to the Jambonz websocket client via `$.refs.client` (attached by the router). ## flowPath Pattern Each node self-selects whether to handle or pass through based on the `flowPath` array in the incoming `tag` data: 1. **No flowPath / empty** (first call, `session:new`): Responds with `answer` + audio verb wrapped in an ack, including a `tag` verb with `flowPath: [nodeId]` and `step_num: 0` 2. **flowPath exists, node NOT in it** (subsequent event): Appends its nodeId to `flowPath`, increments `step_num`, responds with audio verb in ack 3. **flowPath exists, node IS in it** (already visited): Passes message through to Output unchanged In **Say** mode a trailing `gather` verb is appended to trigger the next `session:reconnect`. In **Play** mode the play verb's own `actionHook` triggers the next event. ## Configuration ### Audio Settings - **Audio Type** — Select audio mode: Say (default) or Play - **Say** — Text-to-speech with vendor, language, voice, and loop settings - **Play** — Play an audio file URL with optional timeout, loops, offset, and action hook (default: `/player`) ### Dynamic Value Templates The Prompt and Audio URL fields support template syntax: `{{ $data.fieldName }}` and math: `{{ $data.balance - $data.amount }}` ## Debug Configuration Same debug options as the WS router and gather components.
