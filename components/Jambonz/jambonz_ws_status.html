# Jambonz [WS router] https://docs.jambonz.org/webhooks/websocket-api WebSocket endpoint for Jambonz call routing that correctly negotiates `Sec-WebSocket-Protocol: jambonz.jambonz.org` — on the **same port** as the Flow application. ## How it works Uses `ws` module with `noServer: true` and `handleProtocols` to produce the correct 101 handshake. The component hooks into `F.server`'s `upgrade` event via `prependListener` (runs **before** Total.js) and handles the full WebSocket lifecycle itself. ## Jambonz Messaging Rules Jambonz uses a strict **Ack/Command** pattern over text frames: 1. Jambonz connects and immediately sends a `session:new` JSON payload containing a `msgid`. 2. Your flow **MUST** reply via the **Send** input with a JSON payload where `type` is `ack` and the `msgid` matches. **Example of a valid reply to the Send input:** ```json { "type": "ack", "msgid": "YOUR_RECEIVED_MSGID", "data": [ { "verb": "say", "text": "Hello! The WebSocket is successfully connected." } ] }
