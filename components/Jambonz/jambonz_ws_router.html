# Jambonz [WS router] https://docs.jambonz.org/webhooks/websocket-api WebSocket endpoint for Jambonz call routing that correctly negotiates `Sec-WebSocket-Protocol: jambonz.jambonz.org` — on the **same port** as the Flow application. ## Outputs - **FLOW** — Filtered JSON emitted only for `session:new`, `session:reconnect`, `verb:hook`, and `dial:confirm` event types. Core fields: `type`, `msgid`, `jbz_call_sid`, `direction`, `jbz_app_sid`, `ani`, `dnis`, `jbz_call_id`, `sip_call_id`, `trunk`. When present: `step_num` and `flowPath` (from `data.customerData`). When an ASR result is available (e.g. `verb:hook` from a gather): `asr_confidence` (number) and `asr_transcript` (string, or integer when the transcript is a whole number such as an age). - **raw** — Original message as received, without any transformation (all event types) - **Debug** — Formatted debug/error messages (when "Send to Debug output" options are enabled) ## How it works Uses `ws` module with `noServer: true` and `handleProtocols` to produce the correct 101 handshake. The component hooks into `F.server`'s `upgrade` event via `prependListener` (runs **before** Total.js) and handles the full WebSocket lifecycle itself. ## Maintenance Enable **Enable component debug** to activate diagnostic logging. Options: - **Send Name and ID** — Include component name and unique instance ID in log lines - **Timestamp (ISO 8601)** — Prepend timestamp in the configured timezone - **Timezone** — UTC (default), GMT offsets, or IANA identifiers (with DST) - **JSON output** — Structured JSON format instead of plain text - **Send debug to Debug output** — Route debug messages to the Debug output instead of console - **Send errors to Debug output** — Route error messages to the Debug output instead of console ### Plain text format ` [] ` ### JSON format `{"time":"...","component":"...","nodeId":"[...]","level":"...","message":"..."}` ## Jambonz Messaging Rules Jambonz uses a strict **Ack/Command** pattern over text frames: 1. Jambonz connects and immediately sends a `session:new` JSON payload containing a `msgid`. 2. Your flow **MUST** reply with a JSON payload where `type` is `ack` and the `msgid` matches. Downstream components send responses directly to the WebSocket client via `$.refs.client`.
