# Jambonz Say https://docs.jambonz.org/verbs/verbs/say This component adds a Jambonz `say` verb to the flow for text-to-speech synthesis. ## Configuration - **Prompt** (required): Text to speak (supports static text and dynamic templates) - **Vendor** (optional): TTS vendor to use - **Language** (optional): Language code for TTS - **Voice** (optional): Specific voice to use - **Loop**: Number of times to repeat (default: 1) ## Dynamic Value Templates The Prompt field supports template syntax to read from input data, perform math operations, and concatenate text. ### Example Input Data: ```json { "userName": "John", "accountBalance": 1500, "transactionAmount": 250 } ``` ### Template Syntax: - **Static text**: `Welcome to our service` → outputs `"Welcome to our service"` - **Dynamic path**: `Hello {{ $data.userName }}` → outputs `"Hello John"` - **Text concatenation**: `Your balance is {{ $data.accountBalance }} dollars` → `"Your balance is 1500 dollars"` ### Math Operations: - `Your new balance is {{ $data.accountBalance - $data.transactionAmount }}` → `"Your new balance is 1250"` - `Transaction doubled: {{ $data.transactionAmount * 2 }}` → `"Transaction doubled: 500"` ### Complex Examples: - `Hello {{ $data.userName }}, your account ending in {{ $data.accountNumber }}` → Combines multiple template values - `Balance after transaction: {{ $data.accountBalance - $data.transactionAmount }} dollars` → Math with text ### Error Handling: Unsupported expressions return `""`: - `{{ $data.nonexistent }}` → path doesn't exist - `{{ $data.userName + 5 }}` → cannot add number to string - `{{ $data.account.invalid }}` → invalid nested path ## Behavior - If `jambonz_response` array doesn't exist in input data, creates a new array with the say verb - If `jambonz_response` array exists, appends the say verb to the existing array - Vendor, Language, and Voice are optional - only included if specified - Vendor names are automatically converted to lowercase - Template expressions are resolved before building the say verb - Invalid template expressions return `""` ## Customization dropdown lists To modify dropdown options, edit related arrays in the `
