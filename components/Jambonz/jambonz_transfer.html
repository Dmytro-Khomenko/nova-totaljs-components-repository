# Jambonz Transfer (sip:refer) https://docs.jambonz.org/verbs/verbs/sip-refer This component adds a Jambonz `sip:refer` verb to the flow for transferring calls using SIP REFER method. ## Configuration - **Refer To** (required): The SIP URI or phone number to transfer to (supports templates) - **Referred By** (optional): The SIP URI indicating who is referring the call (supports templates) - **Custom SIP Headers** (optional): Key-value pairs for custom SIP headers - **Event Hook** (optional): URL for event notifications during transfer (supports templates) - **Action Hook** (optional): URL for completion notifications (supports templates) - **Enable hook waiting management**: Configure retry logic for action hook calls ## Dynamic Value Templates All fields (Refer To, Referred By, headers, Event Hook, Action Hook, Define Actions) support template syntax. ### Example Input Data: ```json { "targetNumber": "+15083084809", "callerId": "Customer Service", "webhookBase": "https://example.com", "transferType": "continue" } ``` ### Template Syntax: - **Static value**: `+15083084809` → outputs `"+15083084809"` - **Dynamic path**: `{{ $data.targetNumber }}` → reads from `$.data.targetNumber` → outputs `"+15083084809"` - **Nested path**: `{{ $data.customer.phoneNumber }}` → reads from `$.data.customer.phoneNumber` ### Text Concatenation: - `sip:{{ $data.targetNumber }}@example.com` → `"sip:+15083084809@example.com"` - `{{ $data.webhookBase }}/{{ $data.transferType }}` → `"https://example.com/continue"` - `{{ $data.webhookBase }}/event-notify` → `"https://example.com/event-notify"` - `X-Customer-ID_{{ $data.callerId }}` → `"X-Customer-ID_Customer Service"` ### Error Handling: Unsupported expressions return `""`: - `{{ $data.nonexistent }}` → `""` (path doesn't exist) - `{{ $data.callerId + 5 }}` → `""` (cannot add number to string) - `{{ $data.nested.invalid }}` → `""` (invalid nested path) ## Behavior - If `jambonz_response` array doesn't exist in input data, creates a new array with the transfer verb - If `jambonz_response` array exists, appends the transfer verb to the existing array - Optional fields (referredBy, eventHook, actionHook) are only included if provided - actionHookDelayAction is only added when hook waiting management is enabled and actionHook is provided - Template expressions are resolved before building the transfer verb - Invalid template expressions return `""` - Headers are only added if both key and value are provided
