# Genesys API https://developer.genesys.cloud/devapps/api-explorer This component handles HTTP requests to the Genesys PureCloud API with configurable regions, methods, headers, query parameters, and body. ## Features - **Region Support**: All Genesys PureCloud regions (US, EU, AP, CA, SA, ME) - **HTTP Methods**: GET, POST, PUT, PATCH, DELETE, HEAD - **Dynamic Values**: Template support for paths, headers, query parameters, and body with `{{ }}` syntax - **Query Parameters**: Key-value pairs with template support - **Retry Logic**: Configurable retry attempts for 5xx errors and automatic 429 rate limit handling - **SSL Options**: Option to ignore SSL certificate validation - **Debug Logging**: Separate request and response debugging to server console - **Forward Input Data**: Optional forwarding of input data alongside API response ## Dynamic Value Templates The following fields support template syntax to read from global variables, input data, perform math operations, and concatenate text: - **Path** - **Query Parameters** (both keys and values) - **Headers** (both keys and values) - **Body** ### Template Syntax: - **Global variables**: `{{ $FLOW_VARS.key }}` → reads from global variables (e.g., `{{ $FLOW_VARS.genesys_access_token }}`) - **Static value**: `SomeText` → outputs `"SomeText"` - **Dynamic path**: `{{ $data.key1 }}` → reads from `$.data.key1` - **Nested path**: `{{ $data.key2.key21 }}` → reads from `$.data.key2.key21` - **Array path**: `{{ $data.key3[0].key31 }}` → reads from `$.data.key3[0].key31` ### Examples: - Authorization header: `bearer {{ $FLOW_VARS.genesys_access_token }}` - Query parameter: `pageSize={{ $data.pageSize }}` - Path with variable: `/api/v2/users/{{ $data.userId }}` ### Math Operations: - `{{ $data.key4 + 5 }}` → addition - `{{ $data.key4 - 3 }}` → subtraction - `{{ $data.key4 * 2 }}` → multiplication - `{{ $data.key4 / 2 }}` → division ### Text Concatenation: - `{{ $data.key1 }} test1` → `"value1 test1"` - `prefix_{{ $data.key1 }}_suffix` → `"prefix_value1_suffix"` - `bearer {{ $FLOW_VARS.token }}` → `"bearer abc123..."` ### Error Handling: Unsupported expressions return `""`: - `{{ $FLOW_VARS.nonexistent }}` → variable doesn't exist - `{{ $data.nonexistent }}` → path doesn't exist - `{{ $data.key1 + 5 }}` → cannot add number to string - `{{ $data.key2.invalid }}` → invalid nested path ## Query Parameters Support The component supports query parameters with template support: - **Query Parameters**: Key-value pairs added to the URL query string (supports templates in both keys and values) Query parameters are automatically URL-encoded and appended to the path: - If path has no query string: `?key1=value1&key2=value2` - If path already has query string: `&key1=value1&key2=value2` Template expressions in query parameters: - Support `{{ $FLOW_VARS.* }}` syntax for global variables - Support `{{ $data.* }}` syntax for dynamic values - Both keys and values support templates - Values are automatically URL-encoded ## Additional Options - **Timeout**: Request timeout in milliseconds (default: 10000) - **Max. size**: Response maximum size in bytes (default: 512000) - **Retry attempts**: Number of retries for 5xx errors (default: 0) - **429 Rate Limit Handling**: Automatic single retry using `retry-after` header (ignores configured timeout and retry settings). Adds random 10-30 second buffer to prevent thundering herd. ## Inputs and Outputs - **Input**: Triggers the HTTP request (accepts any data) - **Output**: Returns response wrapped in `genesys_response` object. When "Forward input data" is enabled, also includes `forwarded_data` with the original input data. - **Error**: Returns error details for non-200 responses or request failures ## Output Structure When "Forward input data" is disabled (default): ```json { "genesys_response": { ... API response ... } } ``` When "Forward input data" is enabled: ```json { "forwarded_data": { ... original input data ... }, "genesys_response": { ... API response ... } } ``` ## Behavior - Triggered by any input request - Constructs full URL from region and path - Resolves templates in path, query parameters, headers, and body (supports `$FLOW_VARS` and `$data`) - Applies query parameters to the URL if specified - Applies timeout and size limit settings - Sends HTTP request with configured method, headers, and body - Automatically sets Content-Type to application/json for JSON bodies - Returns parsed JSON response wrapped in `genesys_response` on success (HTTP 200) - Optionally includes `forwarded_data` with original input when enabled - Returns error details with status, headers, and body for failures - **429 Rate Limit**: Automatically retries once after the duration specified in `retry-after` header (in seconds) plus a random 10-30 second buffer to prevent thundering herd - **5xx Errors**: Retries based on configured retry attempts using the configured timeout - Debug logging can be enabled for requests and responses ## Debug Output When "Requests" debug is enabled, the following information is logged to the server console: - URL (with resolved path and query parameters) - HTTP Method - Headers (with Authorization masked for security) - Query Parameters (if configured) - Body (prettified JSON if applicable) - Timeout and limit values - Note if body was configured but not sent (for GET/HEAD requests)
