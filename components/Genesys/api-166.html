# Genesys API https://developer.genesys.cloud/devapps/api-explorer This component handles HTTP requests to the Genesys PureCloud API with configurable regions, methods, headers, query parameters, and body. ## Dynamic Value Templates The following fields support template syntax to read from input data, perform math operations, and concatenate text: - **Path** - **Query Parameters** (both keys and values) - **Headers** (both keys and values) - **Body** ### Template Syntax: - **Static value**: `SomeText` → outputs `"SomeText"` - **Dynamic path**: `{{ $data.key1 }}` → reads from `$.data.key1` - **Nested path**: `{{ $data.key2.key21 }}` → reads from `$.data.key2.key21` - **Array path**: `{{ $data.key3[0].key31 }}` → reads from `$.data.key3[0].key31` ### Math Operations: - `{{ $data.key4 + 5 }}` → addition - `{{ $data.key4 - 3 }}` → subtraction - `{{ $data.key4 * 2 }}` → multiplication - `{{ $data.key4 / 2 }}` → division ### Text Concatenation: - `{{ $data.key1 }} test1` → `"value1 test1"` - `prefix_{{ $data.key1 }}_suffix` → `"prefix_value1_suffix"` ### Error Handling: Unsupported expressions return `""`: - `{{ $data.nonexistent }}` → path doesn't exist - `{{ $data.key1 + 5 }}` → cannot add number to string - `{{ $data.key2.invalid }}` → invalid nested path ## Query Parameters Support The component supports query parameters with template support: - **Query Parameters**: Key-value pairs added to the URL query string (supports templates in both keys and values) Query parameters are automatically URL-encoded and appended to the path: - If path has no query string: `?key1=value1&key2=value2` - If path already has query string: `&key1=value1&key2=value2` Template expressions in query parameters: - Support `{{ $data.* }}` syntax for dynamic values - Both keys and values support templates - Values are automatically URL-encoded ## Additional Options - **Timeout**: Request timeout in milliseconds (default: 10000) - **Max. size**: Response maximum size in bytes (default: 512000) ## Inputs and Outputs - **Input**: Triggers the HTTP request (accepts any data) - **Output**: Returns response body in JSON format for HTTP 200 responses - **Error**: Returns error details for non-200 responses or request failures ## Behavior - Triggered by any input request - Constructs full URL from region and path - Applies query parameters to the URL if specified - Applies timeout and size limit settings - Applies template resolution to path, query parameters, headers, and body - Sends HTTP request with configured method, headers, and body - Automatically sets Content-Type to application/json for JSON bodies - Returns parsed JSON response on success (HTTP 200) - Returns error details with status, headers, and body for failures - Debug logging can be enabled for requests and responses
