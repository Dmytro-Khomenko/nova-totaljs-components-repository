# Genesys OAuth This component handles OAuth authentication with Genesys PureCloud platform. ## Dynamic Value Templates The Client ID and Client Secret fields support template syntax to access global variables and input data: ### Template Syntax: - **Global variables**: `{{ $FLOW_VARS.key }}` → reads from global variables (e.g., `{{ $FLOW_VARS.client_id }}`) - **Dynamic path**: `{{ $data.key1 }}` → reads from `$.data.key1` - **Nested path**: `{{ $data.key2.key21 }}` → reads from `$.data.key2.key21` - **Static value**: `SomeText` → outputs `"SomeText"` ### Examples: - Client ID: `{{ $FLOW_VARS.client_id }}` - Client Secret: `{{ $FLOW_VARS.genesys_client_secret }}` - From data: `{{ $data.credentials.clientId }}` ### Math Operations: - `{{ $data.key4 + 5 }}` → addition - `{{ $data.key4 - 3 }}` → subtraction - `{{ $data.key4 * 2 }}` → multiplication - `{{ $data.key4 / 2 }}` → division ### Text Concatenation: - `{{ $FLOW_VARS.prefix }}_test` → combines global variable with text - `prefix_{{ $data.key1 }}_suffix` → `"prefix_value1_suffix"` ### Error Handling: Unsupported expressions return `""`: - `{{ $FLOW_VARS.nonexistent }}` → variable doesn't exist - `{{ $data.nonexistent }}` → path doesn't exist ## Additional Options - **Timeout**: Request timeout in milliseconds (default: 10000) - **Max. size**: Response maximum size in bytes (default: 512000) - **Retry attempts**: Number of retries for 5xx errors (default: 0) - **429 Rate Limit Handling**: Automatic single retry using `retry-after` header (ignores configured timeout and retry settings) ## Outputs - **Output**: Success response (HTTP 200) with token data - **Error**: Error response for failed authentication or other errors ## Behavior - Component is triggered by any input request - Resolves templates in Client ID and Client Secret (supports `$FLOW_VARS` and `$data`) - Performs HTTP POST request to the selected region's OAuth endpoint - Applies timeout and size limit settings - Sends successful responses to the output - Sends error responses to the error output - Automatically parses JSON responses - **429 Rate Limit**: Automatically retries once after the duration specified in `retry-after` header (in seconds) - **5xx Errors**: Retries based on configured retry attempts using the unified timeout behavior - Debug logging can be enabled for requests and responses
