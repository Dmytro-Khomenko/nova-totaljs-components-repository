# Flow Variables This component manages Flow variables with GET, SET, and DEL operations. Supports multiple operations in a single execution. ## Inputs and Outputs - **Input**: Triggers the variable operations (accepts any data) - **Output**: Returns combined results in JSON format, always wrapped in `flow_variables_response` ## Variables Management You can add multiple variable operations that will be executed sequentially: - **Action**: Operation to perform (GET, SET, DEL) - **Variable**: Name of the flow variable (mandatory, supports templates) - **Value**: Value to set (visible only for SET action, supports templates) - **Type**: Return type for GET operation (visible only for GET action: String, Number, Boolean) ## Template Support Both **Variable** and **Value** fields support dynamic templates with `{{ $data.* }}` syntax. ### Template Syntax: - **Static value**: `myVar` → outputs `"myVar"` - **Dynamic path**: `{{ $data.keyName }}` → reads from `$.data.keyName` - **Nested path**: `{{ $data.nested.value }}` → reads from `$.data.nested.value` - **Array path**: `{{ $data.items[0].name }}` → reads from `$.data.items[0].name` - **Object/Array**: `{{ $data.body }}` → automatically converted to JSON string ### Math Operations: - `{{ $data.value + 5 }}` → addition - `{{ $data.value - 3 }}` → subtraction - `{{ $data.value * 2 }}` → multiplication - `{{ $data.value / 2 }}` → division ### Text Concatenation: - `{{ $data.prefix }}_key` → `"value_key"` - `prefix_{{ $data.value }}_suffix` → `"prefix_value_suffix"` ### Error Handling: Unsupported expressions return `""`: - `{{ $data.nonexistent }}` → path doesn't exist - `{{ $data.text + 5 }}` → cannot add number to string - `{{ $data.nested.invalid }}` → invalid nested path ## Multiple Operations The component processes all configured rows sequentially. You can mix GET, SET, and DEL operations in a single execution: **Example Configuration**: 1. Row 1: GET "counter" as Number 2. Row 2: SET "lastAccess" to "{{ $data.timestamp }}" 3. Row 3: DEL "tempData" ## Behavior Notes - Variables are stored at the Flow level and persist across component executions - Variable names are case-sensitive - All responses are wrapped in `flow_variables_response` object - GET operation applies type conversion to returned values based on the selected Type - Number conversion uses `parseFloat()` - if conversion fails, original value is returned - Boolean conversion for strings recognizes: 'true', '1', 'yes' (case-insensitive) as `true` - Template expressions are resolved before operations are performed - All rows are processed sequentially in the order they appear - Operations that modify variables (SET/DEL) trigger `instance.newvariables()` to notify the system - When "Forward input data" is enabled, the response includes both `forwarded_data` and `flow_variables_response` - Forwarding is useful for passing data through the component while also performing variable operations - This component operates on `instance.main.variables` which are Flow-level variables For more information about Flow variables, see: https://docs.totaljs.com/flowstream/d27dc002rl50c/#d27e2001rl51c
