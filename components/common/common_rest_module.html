$NAME
### REST module A flexible HTTP client component for making REST API requests with dynamic template support. ## Features - **HTTP Methods**: GET, POST, PUT, PATCH, DELETE, HEAD - **Dynamic Templates**: Support for `{{ $FLOW_VARS.* }}` and `{{ $data.* }}` in URL, headers, query parameters, and body - **Response Configuration**: Choose which parts of the response to return (status code, headers, body) - **Data Forwarding**: Preserve original input data and optionally combine responses when chaining multiple REST modules - **Retry Logic**: Configurable retry attempts for 5xx errors - **SSL Options**: Option to ignore SSL certificate validation - **Debug Logging**: Separate request and response debugging to server console ## Dynamic Template Syntax The following fields support template variables: - **URL**: Full URL with dynamic values - **Query Parameters**: Both keys and values support templates - **Headers**: Both keys and values support templates - **Body**: Full body content with template support ### Template Examples: - `{{ $FLOW_VARS.api_token }}` - Read from global variables - `{{ $data.user_id }}` - Read from input data - `{{ $data.items[0].name }}` - Access nested arrays and objects - `{{ $data.count + 5 }}` - Math operations (+, -, *, /) ### Usage in Body: ```json { "userId": "{{ $data.user_id }}", "token": "{{ $FLOW_VARS.api_token }}", "embeddings": [{{ $data.embeddings[0] }}], "count": {{ $data.count + 1 }} } ``` ## Response Configuration Select which parts of the HTTP response to include: - **Status code**: HTTP status code (e.g., 200, 404, 500) - **Headers**: Response headers object - **Body**: Response body (automatically parsed if JSON) ## Output Structure Default (Body only): ```json { "rest_response": { "body": { ... } } } ``` With all response options enabled: ```json { "rest_response": { "status": 200, "headers": { ... }, "body": { ... } } } ``` With "Forward input data" enabled: ```json { "forwarded_data": { ... original input ... }, "rest_response": { ... } } ``` With both "Forward input data" and "Forward response" enabled (chaining multiple REST modules): ```json { "forwarded_data": { ... original input ... }, "rest_response": [ { ... first module response ... }, { ... second module response ... } ] } ``` ## Data Forwarding Behavior - **Forward input data**: Preserves the original input data through the chain of REST modules - First module: Creates `forwarded_data` from input - Subsequent modules: Maintains the same `forwarded_data` without nesting - **Forward response**: Combines responses from multiple REST modules into an array - Enables accessing previous responses: `$data.rest_response[0]`, `$data.rest_response[1]`, etc. - Useful for chaining API calls where later calls need data from earlier responses ## Additional Options - **Timeout**: Request timeout in milliseconds (default: 10000) - **Max. size**: Response maximum size in bytes (default: 512000) - **Retry attempts**: Number of retries for 5xx errors (default: 0) - **Ignore SSL**: Skip SSL certificate validation ## Debug Output - **Requests**: Logs URL, method, headers (with auth masked), query params, and body - **Responses**: Logs status code, headers, and response body - Debug prints to server-side Node.js log or docker log output (not browser console)
