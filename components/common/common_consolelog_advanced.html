# console.log(Advanced) Enhanced console logging component with configurable output formatting. ## Features - **Timestamp (ISO 8601)**: Adds ISO 8601 formatted timestamp to each log entry - **Timezone with dropdown**: Supports both GMT offsets (fixed) and IANA timezone identifiers (with automatic DST) - **Source component**: Displays the name of the component sending the message - **JSON output**: When enabled, wraps all output in structured JSON format for easy parsing - **Array iteration**: Arrays are automatically iterated, logging each element separately ## Configuration All options are configurable in the component settings: - **Timestamp (ISO 8601)** - Default: `true` - Include timestamp in output - **Timezone** - Default: `UTC` - Supports both IANA timezone identifiers and GMT offset syntax - **Source component** - Default: `true` - Include source component name - **JSON output** - Default: `false` - Wrap output in structured JSON format ## Output Modes ### Plain Text Mode (JSON output = false) Human-readable format with inline JSON. Best for visual inspection of logs. - Format: ` component: message: ` - All content on single lines (no multi-line formatting) - Arrays are iterated, each element logged separately - Objects are inline JSON strings ### JSON Output Mode (JSON output = true) Structured JSON format. Best for log processing tools and parsers (e.g., Logstash, Fluentd). - Format: `{"timestamp": "...", "component": "...", "message": ...}` - Each log line is a complete, parseable JSON object - Arrays are iterated, each element in its own JSON wrapper - Message content preserved as native JSON types (not stringified)
