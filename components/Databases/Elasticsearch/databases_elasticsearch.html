# Elasticsearch [Docs] This component provides integration with Elasticsearch for document operations via REST API. ## Configuration ### Request settings - **Operation:** Select the Elasticsearch operation to perform - **Get:** Retrieve a single document by ID (requires Doc Id) - **Get many:** Retrieve multiple documents (no Doc Id field) - **Exists:** Check if a document exists (requires Doc Id, returns HEAD response) - **Index:** Create a new document with specified ID (requires Doc Id) - **Delete:** Delete a document by ID (requires Doc Id) - **Update:** Update an existing document (requires Doc Id) - **Search by query:** Search for documents matching a query (no Doc Id field) - **Delete by query:** Delete documents matching a query (no Doc Id field) - **Update by query:** Update documents matching a query (no Doc Id field) - **Index:** Elasticsearch index name (required, supports templates) - **Doc Id:** Document ID (required for Get, Exists, Index, Delete, Update operations; hidden for Get many, Search by query, Update by query, Delete by query operations; supports templates) - **Query Parameters:** Additional URL query parameters as key-value pairs - **Headers:** Custom HTTP headers as key-value pairs ### Request body - **Body:** JSON body for operations that support it (Index, Update, Get many, Search by query, Update by query, Delete by query) - **Example:** Dynamic visual reference showing operation-specific examples and template usage (changes based on selected operation) ### Additional Options - **Max. size:** Response maximum size in bytes (default: 512000) - **Timeout:** Request timeout in milliseconds (default: 10000) - **Retry attempts:** Number of retry attempts if request fails (default: 0). If set to a value greater than 0, the component will retry the request after waiting for the timeout duration between each attempt. - **Ignore SSL:** Skip SSL certificate validation for HTTPS connections ## Template Support All Instance settings, Request settings fields (Index, Doc Id, Query Parameters, Headers), and Request body support template syntax: ### Template Syntax: - **Flow variables:** `{{ $FLOW_VARS.variable_name }}` → reads from flow variables - **Full payload:** `{{ $data }}` → entire input data as JSON string - **Input data path:** `{{ $data.key }}` → reads from `$.data.key` - **Nested path:** `{{ $data.nested.value }}` → reads from `$.data.nested.value` ### Advanced Template Examples: **Using {{ $data(objArray) }} for bulk operations:** Input data (array of objects): ```json { "data": [ { "field1": "value1", "field2": 123 }, { "field1": "value2", "field2": 456 } ] } ``` Body with `{{ $data(objArray) }}`: Results in: ```json { "field1": "value1", "field2": 123 }, { "field1": "value2", "field2": 456 } ``` ## Outputs - **Output:** Successful response (HTTP 2xx) with parsed JSON body - For Exists operation: `{ exists: true }` or `{ exists: false }` - **Error:** Error response with status, headers, and body details ## Behavior - Component is triggered by any input request - Resolves all template expressions before executing the request - Automatically adds Basic Authentication header when enabled - Parses JSON responses automatically - Sends successful responses (2xx) to output - Sends error responses to error output - Debug logging can be enabled for detailed request/response information - Retry logic: If retry attempts > 0, the component will automatically retry failed requests (network errors or non-2xx HTTP status codes). It waits for the timeout duration between each retry attempt. For example, with retry attempts = 3 and timeout = 10000ms, the component will make up to 4 total attempts (1 initial + 3 retries), waiting 10 seconds between each attempt.
