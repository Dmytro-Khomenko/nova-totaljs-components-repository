Additional description:
+-----------+-------------------------------------------------------------------------------------------------+
|  Command  |  Description                                                                                    |
+-----------+-------------------------------------------------------------------------------------------------+
|  LENGTH   |  Returns the length of the list stored at key.                                                  |
|           |  If key does not exist, it is interpreted as an empty list and 0 is returned.                   |
|           |  An error is returned when the value stored at key is not a list.                               |
+-----------+-------------------------------------------------------------------------------------------------+
|  RANGE    |  Returns the specified elements of the list stored at key.                                      |
|           |  The offsets start and stop are zero-based indexes, with 0 being the first element of the list. |
|           |  The offsets can also be negative numbers indicating offsets starting at the end of the list.   |
|           |  For example, -1 is the last element of the list, -2 the penultimate, and so on.                |
+-----------+-------------------------------------------------------------------------------------------------+
|  PUSH     |  Insert value or values at the head/tail of the list stored at key.                             |
|           |  If key does not exist, it is created as empty list before performing the push operations.      |
+-----------+-------------------------------------------------------------------------------------------------+
|  POP      |  Returns the first/last element and also removes it from the list stored at key .               |
|           |  When provided with the count argument, the reply will consist the count of elements.           |
+-----------+-------------------------------------------------------------------------------------------------+

# Redis This component provides integration with Redis database instances, supporting various operations for data storage and retrieval. ## Inputs and Outputs - **Input**: Triggers the Redis operation (accepts any data) - **Output**: Returns the operation result wrapped in `redis_response` object (or error object if operation fails) ## Behavior Notes - Component requires the `redis` npm package - Component is triggered by any input request - Establishes connection to Redis instance with configured settings - Returns the operation result wrapped in `redis_response` object - Automatically handles connection management and error handling - Template expressions are resolved before executing Redis commands - Invalid template expressions return `""` - Connection is reused across multiple operations ### Templates supported syntax: - **Static value**: `mykey` → outputs `"mykey"` - **Dynamic path**: `{{ $data.keyName }}` → reads from `$.data.keyName` - **Nested path**: `{{ $data.nested.value }}` → reads from `$.data.nested.value` - **Array path**: `{{ $data.items[0].name }}` → reads from `$.data.items[0].name` - **Object/Array**: `{{ $data.body }}` → automatically converted to JSON string ### Templates Math Operations: - `{{ $data.value + 5 }}` → addition - `{{ $data.value - 3 }}` → subtraction - `{{ $data.value * 2 }}` → multiplication - `{{ $data.value / 2 }}` → division ### Templates Text Concatenation: - `{{ $data.prefix }}_key` → `"value_key"` - `prefix_{{ $data.value }}_suffix` → `"prefix_value_suffix"` ### Templates Error Handling: Unsupported expressions return `""`: - `{{ $data.nonexistent }}` → path doesn't exist - `{{ $data.text + 5 }}` → cannot add number to string - `{{ $data.nested.invalid }}` → invalid nested path
